1. Introduction to TypeScript

What is TypeScript?

TypeScript is a statically typed superset of JavaScript developed and maintained by Microsoft. It adds optional static typing and class-based object-oriented programming to JavaScript. With TypeScript, you can catch errors during development rather than at runtime, making your code more robust and maintainable.

Key Differences from JavaScript

  • Static typing with type annotations
  • Interfaces and type declarations
  • Advanced object-oriented features (e.g., classes, interfaces, generics)
  • Generics for creating reusable components
  • Enhanced IDE support with better autocompletion and error checking

TypeScript vs JavaScript Example

JavaScript

// JavaScript
function add(a, b) {
  return a + b;
}

// This works
console.log(add(5, 3));  // 8

// This also "works" but might not be what you want
console.log(add("5", 3)); // "53"

TypeScript

// TypeScript
function add(a: number, b: number): number {
  return a + b;
}

// This works
console.log(add(5, 3));  // 8

// This will show a compile-time error
// console.log(add("5", 3));
// Argument of type 'string' is not 
// assignable to parameter of type 'number'.

Notice how TypeScript prevents you from accidentally passing a string where a number is expected, catching potential bugs before they happen.

When to Use TypeScript

  • Large applications: Type safety becomes more valuable as your codebase grows.
  • Team projects: Types serve as documentation and improve collaboration.
  • Complex domain models: Types help represent business logic more clearly.
  • Applications requiring high reliability: Catch more errors before deployment.

Part B: Project Setup

Instructions: Create a new TypeScript project with the following steps:

  1. Create a new directory called `my-ts-project`.
  2. Navigate to this directory in your terminal.
  3. Initialize a new npm project with default settings (run `npm init -y`).
  4. Install TypeScript as a development dependency (run `npm install --save-dev typescript`).
  5. Create a `tsconfig.json` file (run `npx tsc --init`).
  6. Create the following directory structure: 
    my-ts-project/
├── src/
│   └── index.ts  // Create an empty index.ts file here
├── dist/         // This will be created by the compiler
├── package.json
└── tsconfig.json
    (You'll need to manually create the `src` folder and the `index.ts` file inside it.)
  7. Edit your `tsconfig.json` to specify:
    • Target: `"es6"` (or a more recent version like `"es2020"`)
    • Module: `"commonjs"`
    • Root directory: `"rootDir": "./src"`
    • Output directory: `"outDir": "./dist"`
    Ensure these options are uncommented and set correctly within the `compilerOptions` object in `tsconfig.json`.
Expected Outcome & Solution Guidance

After completing these steps, you should have a `my-ts-project` directory containing a `package.json`, a `tsconfig.json` configured as specified, and an `src` folder with an empty `index.ts` file. The `dist` folder will be created when you first compile your project (e.g., by running `tsc` in the project root).

Example `tsconfig.json` relevant parts:

{
  "compilerOptions": {
    "target": "es6",                     /* Specify ECMAScript target version */
    "module": "commonjs",                /* Specify module code generation */
    "outDir": "./dist",                  /* Redirect output structure to the directory. */
    "rootDir": "./src",                  /* Specify the root directory of input files. */
    "strict": true,                      /* Enable all strict type-checking options. */
    "esModuleInterop": true,             /* Enables emit interoperability between CommonJS and ES Modules via creation of namespace objects for all imports. Implies 'allowSyntheticDefaultImports'. */
    "skipLibCheck": true,                /* Skip type checking of declaration files. */
    "forceConsistentCasingInFileNames": true /* Disallow inconsistently-cased references to the same file. */
  },
  "include": ["src/**/*"],              /* Specifies an array of filenames or patterns to include in the program. */
  "exclude": ["node_modules", "dist"]   /* Specifies an array of filenames or patterns that should be skipped when resolving include. */
}

To verify, you can add `console.log("Hello from index.ts");` to `src/index.ts`, run `tsc` in the terminal (while in the `my-ts-project` directory), and then run `node dist/index.js`. You should see "Hello from index.ts" printed.

Tasks

Part A: Creating and Customizing `tsconfig.json`

Instructions: Create and customize a TypeScript configuration file:

  1. If you haven't already from Exercise 2, ensure you are in your `my-ts-project` directory. If `tsconfig.json` doesn't exist, create it: 
    npx tsc --init
  2. Modify your `tsconfig.json` to include the following settings within `compilerOptions`:
    • `"target": "es6"`
    • `"module": "commonjs"`
    • `"rootDir": "./src"`
    • `"outDir": "./dist"`
    • `"strict": true`
    • `"noImplicitAny": true`
    • `"sourceMap": true`
    (Ensure these lines are uncommented and correctly set.)
  3. Create a simple TypeScript file in your `src` directory (e.g., `src/app.ts`) with some basic code: `console.log("App is running!");`
  4. Compile your project by running `tsc` in the terminal from your project root.
  5. Verify that the compiled JavaScript (`app.js`) and its source map (`app.js.map`) appear in the `dist` directory.

Part B: Experimenting with Compiler Options

Instructions: Experiment with different compiler options:

  1. Create a file called `experiment.ts` in your `src` directory with the following code: 
    // src/experiment.ts
// Add a comment here
function calculateArea(width, height) { // Parameters 'width' and 'height' implicitly have an 'any' type.
  return width * height;
}

let result = calculateArea(5, 10);
console.log(\`The area is \${result}\`);
  2. Try compiling with different settings in `tsconfig.json` (run `tsc` after each change):
    • Set `"noImplicitAny": false`, then compile. Observe any changes in compiler messages. Then set it back to `true` and compile. Notice the error about implicit 'any' types for `width` and `height`. Fix this by adding `number` types to the parameters: `function calculateArea(width: number, height: number)`.
    • Set `"target": "es5"`, compile, and look at `dist/experiment.js`. Then change to `"target": "es2020"`, compile, and compare the generated JavaScript. Notice differences in syntax (e.g., `var` vs `let/const`, arrow functions if used).
    • Add some comments to your `experiment.ts`. Then set `"removeComments": true` in `tsconfig.json`, compile, and check if comments are stripped from `dist/experiment.js`. Set it back to `false` or remove the line.
  3. For each change, observe the differences in compiler warnings/errors and the generated JavaScript code in the `dist` folder.
Expected Outcome & Solution Guidance

You should have a `tsconfig.json` file configured as requested in Part A. After running `tsc`, your `src/app.ts` should compile into `dist/app.js` and `dist/app.js.map`.

For Part B:

  • `noImplicitAny`: When `true`, TypeScript will error if it infers a type as `any` because no explicit type was provided. You'll need to add type annotations (e.g., `width: number`). When `false`, it allows implicit `any` types.
  • `target`: This option changes the ECMAScript version of the output JavaScript. `es5` produces older, more widely compatible JavaScript. `es2020` produces modern JavaScript with newer features. You'll see syntax differences like `var` (ES5) vs. `let`/`const` (ES6+).
  • `removeComments`: When `true`, comments are removed from the compiled JavaScript output. When `false` (or not set), comments are typically preserved.

The key is to observe how these settings influence the compiler's behavior and the resulting JavaScript, helping you tailor the compilation process to your project's needs.

Part A (`union_types.ts` - `handleApiResponse` example):

type SuccessResponse = { success: true; data: unknown; };
type ErrorResponse = { success: false; error: { code: number; message: string; }; };
type ApiResponse = SuccessResponse | ErrorResponse;

function handleApiResponse(response: ApiResponse): void {
  if (response.success) { // Type guard narrows to SuccessResponse
    console.log(\`Data received: \${JSON.stringify(response.data)}\`);
  } else { // Type guard narrows to ErrorResponse
    console.log(\`Error \${response.error.code}: \${response.error.message}\`);
  }
}

Part B (`intersection_types.ts`): Intersection types combine all properties of the constituent types.

Part C (`literal_types.ts` - `logData` example):

type LogLevel = "debug" | "info" | "warn" | "error";
function logData(message: string, level: LogLevel): void {
  console.log(\`[\${level.toUpperCase()}]: \${message}\`);
}
logData("User logged in.", "info");
// logData("System critical.", "critical"); // Error: Argument of type '"critical"' is not assignable to parameter of type 'LogLevel'.

interface UserProfile { /* ... as defined ... */ }

  return profile.preferences?.theme ?? "light";
