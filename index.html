Introduction to TypeScript

What is TypeScript?

  • A superset of JavaScript developed by Microsoft
  • Adds static typing to JavaScript
  • Compiles down to plain JavaScript
  • Open source language

Why Use TypeScript?

  • Catch errors earlier (at compile time vs. runtime)
  • Better code completion and IntelliSense
  • Improved code maintainability for larger projects
  • Better documentation through type annotations
  • Enhanced refactoring capabilities

Example: JavaScript vs TypeScript

JavaScript:


function add(a, b) {
  return a + b;
}

// These all work, but some might not be what you intended
console.log(add(5, 3));         // 8
console.log(add("5", 3));       // "53"
console.log(add(true, false));  // 1

TypeScript:


function add(a: number, b: number): number {
  return a + b;
}

console.log(add(5, 3));         // 8
// console.log(add("5", 3));    // Error: Argument of type 'string' is not assignable to parameter of type 'number'
// console.log(add(true, false)); // Error: Argument of type 'boolean' is not assignable to parameter of type 'number'

Note: TypeScript errors are shown as comments to prevent runtime errors in a JS environment if this code were run directly.

Your First TypeScript Program

Creating a TypeScript File

  • Create a file with .ts extension
  • Example: hello.ts

Basic Syntax

  • TypeScript syntax is JavaScript syntax with type annotations
  • Type annotations use : followed by the type

Example: Hello TypeScript


// hello.ts
function greet(name: string): string {
  return `Hello, ${name}!`;
}

const userName: string = "TypeScript";
console.log(greet(userName));

Compiling TypeScript

  • Use the TypeScript compiler (tsc):

tsc hello.ts
  • This creates hello.js that can run in any JavaScript environment
  • Run the compiled code:

node hello.js

Example: Compilation Output

TypeScript (hello.ts):


function greet(name: string): string {
  return `Hello, ${name}!`;
}

const userName: string = "TypeScript";
console.log(greet(userName));

Compiled JavaScript (hello.js):


function greet(name) {
  return "Hello, " + name + "!";
}

var userName = "TypeScript";
console.log(greet(userName));

TypeScript Compiler Configuration

The tsconfig.json File

  • Central configuration file for TypeScript projects
  • Created with tsc --init
  • Controls compiler behavior and options

Key Configuration Options

  • target: JavaScript version to compile to (e.g., ES5, ES6)
  • module: Module system (CommonJS, ES6, etc.)
  • rootDir: Source files location
  • outDir: Compiled output location
  • strict: Enable strict type checking
  • noImplicitAny: Raise error on expressions with implied 'any' type

Example: Basic tsconfig.json


{
  "compilerOptions": {
    "target": "ES6",
    "module": "CommonJS",
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true,
    "noImplicitAny": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

Compiling with tsconfig.json

  • When tsconfig.json exists, simply run:

tsc
  • TypeScript will use the configuration options automatically
  • Watch mode for automatic recompilation:

tsc --watch

Basic Types in TypeScript

Primitive Types

  • number: All numeric values (integer and floating-point)
  • string: Text values
  • boolean: true/false values

Example: Primitive Types


// Number type
let age: number = 25;
let price: number = 19.99;
let hexValue: number = 0xFF; // 255 in decimal

// String type
let firstName: string = "John";
let greeting: string = `Hello, ${firstName}!`;

// Boolean type
let isActive: boolean = true;
let hasPermission: boolean = false;

Arrays

  • Typed collections of elements
  • Two syntax options:
    • type[]
    • Array<type>

Example: Arrays


// Number array
let scores: number[] = [85, 92, 78, 90];
let grades: Array<number> = [3.8, 3.9, 4.0];

// String array
let names: string[] = ["Alice", "Bob", "Charlie"];

// Mixed arrays are not allowed with specific types
// This will cause an error:
// let mixed: number[] = [1, "two", 3]; // Error!

Note: The error line is commented out to ensure the example is illustrative without breaking in a pure JS context.

Tuples

  • Fixed-length arrays with elements of specific types
  • Order and types are important

Example: Tuples


// Define a tuple type: [string, number]
let person: [string, number] = ["Alice", 25];

// Accessing tuple elements
console.log(person[0]); // "Alice"
console.log(person[1]); // 25

// Error: Type mismatch
// person = [25, "Alice"]; // Error: Type 'number' is not assignable to type 'string'

// Error: Too many elements
// person = ["Alice", 25, true]; // Error: Source has 3 elements but target allows only 2

Note: Error lines are commented for illustration.

Enums

  • Named constants for related values
  • Makes code more readable and maintainable

Example: Enums


// Numeric enum (default starts at 0)
enum Direction {
  North,
  East,
  South,
  West
}

let myDirection: Direction = Direction.North;
console.log(myDirection); // 0

// String enum
enum HttpStatus {
  OK = "OK",
  NotFound = "NOT_FOUND",
  ServerError = "SERVER_ERROR"
}

let status: HttpStatus = HttpStatus.OK;
console.log(status); // "OK"

Type Inference

  • TypeScript can automatically determine types
  • Explicit type annotations not always needed

Example: Type Inference


// Type inference for primitives
let message = "Hello"; // TypeScript infers type as string
// message = 42; // Error: Type 'number' is not assignable to type 'string'

// Type inference for arrays
let numbers = [1, 2, 3]; // TypeScript infers type as number[]
// numbers.push("four"); // Error: Argument of type 'string' is not assignable to parameter of type 'number'

Note: Error lines are commented for illustration.

Functions in TypeScript

Function Type Annotations

  • Parameter types
  • Return type
  • Optional and default parameters

Example: Basic Function Types


// Function with parameter and return type annotations
function multiply(a: number, b: number): number {
  return a * b;
}

// Arrow function with type annotations
const divide = (a: number, b: number): number => {
  return a / b;
};

// Void return type (function doesn't return a value)
function logMessage(message: string): void {
  console.log(message);
}

Optional Parameters

  • Marked with ? after parameter name
  • Must come after required parameters

Example: Optional Parameters


// Function with optional parameter
function buildName(firstName: string, lastName?: string): string {
  if (lastName) {
    return `${firstName} ${lastName}`;
  } else {
    return firstName;
  }
}

// Valid calls
console.log(buildName("John", "Doe"));  // "John Doe"
console.log(buildName("John"));         // "John"

Default Parameters

  • Provide default values for parameters
  • Used when argument is not provided or undefined

Example: Default Parameters


// Function with default parameter
function greet(name: string, greeting: string = "Hello"): string {
  return `${greeting}, ${name}!`;
}

// Valid calls
console.log(greet("John", "Hi"));     // "Hi, John!"
console.log(greet("John"));           // "Hello, John!"

Function Types

  • Describe the signature of a function
  • Can be used for variables, parameters, and properties

Example: Function Types


// Define a function type
type MathOperation = (x: number, y: number) => number;

// Use the function type for a variable
let add: MathOperation = (a, b) => a + b;
let subtract: MathOperation = (a, b) => a - b;

// Use the function type as a parameter
function calculate(a: number, b: number, operation: MathOperation): number {
  return operation(a, b);
}

// Call the function
console.log(calculate(10, 5, add));      // 15
console.log(calculate(10, 5, subtract)); // 5

Objects and Interfaces in TypeScript

Object Type Annotations

  • Define the shape of objects
  • Specify property types
  • Can include optional and readonly properties

Example: Object Type Annotations


// Object with inline type annotation
let person: { name: string; age: number } = {
  name: "Alice",
  age: 30
};

// Accessing properties
console.log(person.name); // "Alice"
console.log(person.age);  // 30

// Error: Property doesn't exist
// console.log(person.email); // Error: Property 'email' does not exist on type '{ name: string; age: number }'

Note: Error line is commented for illustration.

Type Aliases

  • Create reusable type definitions
  • Improve code readability
  • Can be used for any type, not just objects

Example: Type Aliases


// Define a type alias for a person object
type Person = {
  name: string;
  age: number;
  email?: string; // Optional property
};

// Use the type alias
let student: Person = {
  name: "Bob",
  age: 22
};

let teacher: Person = {
  name: "Dr. Smith",
  age: 45,
  email: "smith@university.edu"
};

Readonly Properties

  • Prevent modification of properties after initialization
  • Enforced at compile time, not runtime

Example: Readonly Properties


// Object type with readonly property
type Point = {
  readonly x: number;
  readonly y: number;
};

// Create a point
let origin: Point = { x: 0, y: 0 };

// Error: Cannot modify readonly property
// origin.x = 10; // Error: Cannot assign to 'x' because it is a read-only property

Note: Error line is commented for illustration.

Interfaces

  • Similar to type aliases for objects
  • Can be extended and implemented
  • Used primarily for object shapes

Example: Interfaces


// Define an interface
interface Vehicle {
  make: string;
  model: string;
  year: number;
}

// Use the interface
let car: Vehicle = {
  make: "Toyota",
  model: "Corolla",
  year: 2020
};

// Extending interfaces
interface ElectricVehicle extends Vehicle {
  batteryCapacity: number;
  range: number;
}

// Use the extended interface
let tesla: ElectricVehicle = {
  make: "Tesla",
  model: "Model 3",
  year: 2022,
  batteryCapacity: 75,
  range: 310
};

Advanced Types in TypeScript

Union Types

  • Allow a value to be one of several types
  • Use the | operator between types

Example: Union Types


// Variable that can be either string or number
let id: string | number;

id = "abc123";  // Valid
id = 456;       // Valid
// id = true;      // Error: Type 'boolean' is not assignable to type 'string | number'

// Function that accepts union type
function printId(id: string | number) {
  console.log(`ID: ${id}`);
  
  // Type narrowing
  if (typeof id === "string") {
    // In this block, TypeScript knows id is a string
    console.log(id.toUpperCase());
  } else {
    // In this block, TypeScript knows id is a number
    console.log(id.toFixed(2));
  }
}
printId("testID");
printId(123.456);

Note: Error line is commented for illustration.

Intersection Types

  • Combine multiple types into one
  • Use the & operator between types
  • Must satisfy all the combined types

Example: Intersection Types


// Define two types
type Employee = {
  id: number;
  name: string;
};

type Manager = {
  department: string;
  reports: number;
};

// Create an intersection type
type ManagerWithEmployeeInfo = Employee & Manager;

// Use the intersection type
let seniorManager: ManagerWithEmployeeInfo = {
  id: 12345,
  name: "Jane Smith",
  department: "Engineering",
  reports: 5
};

// Error: Missing properties
/*
let juniorManager: ManagerWithEmployeeInfo = {
  id: 67890,
  name: "John Doe"
  // Error: Property 'department' is missing
  // Error: Property 'reports' is missing
};
*/

Note: Error block is commented for illustration.

Literal Types

  • Specific values (not just types)
  • Useful for restricting values to a set of options

Example: Literal Types


// String literal type
type Direction = "north" | "south" | "east" | "west";

function move(direction: Direction) {
  console.log(`Moving ${direction}`);
}

move("north");  // Valid
// move("up");     // Error: Argument of type '"up"' is not assignable to parameter of type 'Direction'

// Numeric literal type
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

function rollDice(): DiceRoll {
  return Math.floor(Math.random() * 6) + 1 as DiceRoll;
}
console.log("Rolled a " + rollDice());

Note: Error line is commented for illustration.

Nullable Types

  • Handle null and undefined values
  • Use union types with null/undefined
  • Requires strictNullChecks compiler option (assumed enabled for these examples)

Example: Nullable Types


// With strictNullChecks enabled (conceptual example)
let nameStrict: string;
nameStrict = "Alice";
// nameStrict = null;      // Error: Type 'null' is not assignable to type 'string'
// nameStrict = undefined; // Error: Type 'undefined' is not assignable to type 'string'

// Explicitly allow null or undefined
let username: string | null | undefined;
username = "bob123";
username = null;      // Valid
username = undefined; // Valid

Note: Error lines are commented for illustration. Actual behavior depends on tsconfig.json.

Optional Property Access Operator

  • Safely access properties that might be null or undefined
  • Use the ?. operator
  • Returns undefined if the object is null or undefined

Example: Optional Property Access


// Define user type for clarity
type User = {
    name: string;
    address?: { city: string };
};

// Original function for comparison (if needed, but direct use of ?. is clearer)
/*
function getUserCityWithoutOptionalChaining(user: User) {
  if (user.address) {
    return user.address.city;
  }
  return undefined;
}
*/

// With optional chaining
function getUserCity(user: User): string | undefined {
  return user.address?.city;
}

// Example usage
const user1: User = {
  name: "Alice",
  address: { city: "New York" }
};

const user2: User = {
  name: "Bob"
  // No address property
};

console.log(getUserCity(user1)); // "New York"
console.log(getUserCity(user2)); // undefined
